# Workflow Rules

## Rule 1: Publish completed work immediately

When work is done, do not leave it only on the local machine.

Always:

1. Commit the finished work.
2. Push the branch to GitHub.
3. Open a pull request for review and merge.

## Why this rule exists

- Keeps GitHub up to date with local changes.
- Prevents the local checkout from drifting too far ahead of the remote.
- Reduces avoidable merge conflicts.
- Makes work visible, recoverable, and easier to review.

## Expected behavior

Work is not considered complete until the code and docs are published to GitHub.
